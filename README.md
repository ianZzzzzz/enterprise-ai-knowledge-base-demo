# Enterprise AI Knowledge Base Demo

企业 AI 知识库 Demo，用于验证 Linear + GitHub + Codex 的协作开发流程，并为后续构建企业文档上传、检索增强生成（RAG）、问答和前端交互能力提供清晰的项目骨架。

## 项目目标

本仓库当前阶段只初始化项目结构，不实现复杂业务逻辑。后续目标包括：

- 支持企业知识文档的上传与管理。
- 支持文档解析、切分、向量化和检索。
- 支持基于知识库上下文的 AI 问答。
- 提供简洁的前端界面，用于上传文档、发起检索和查看回答。
- 保持前后端、文档、测试和脚本目录职责清晰，方便逐步迭代。

## 技术栈建议

当前不引入具体框架依赖，后续可按 issue 分阶段选择：

- 后端：Python FastAPI 或 Node.js NestJS / Express。
- 前端：React / Next.js / Vite。
- 检索：向量数据库或本地向量索引，例如 pgvector、Qdrant、Chroma 等。
- AI 能力：OpenAI API 或兼容 OpenAI 接口的模型服务。
- 测试：后端单元测试、前端组件测试和端到端基础验证。

## 本地开发方式

当前仓库只包含基础目录结构，因此本地启动服务的命令会在后续实现后补充。现阶段建议：

1. 克隆仓库。
2. 阅读 `README.md` 了解项目目标和目录约定。
3. 按后续 issue 分别在 `backend/`、`frontend/`、`docs/`、`tests/` 和 `scripts/` 中添加代码或文档。
4. 在引入具体框架前，优先保持目录清晰、变更小、便于评审。

## 开发计划

建议按以下阶段推进：

1. 后端基础 API：创建健康检查接口和基础配置管理。
2. 文档上传：支持本地上传文件并保存元数据。
3. 文档解析：支持基础文本解析、切分和索引准备。
4. 检索能力：接入向量索引并提供检索 API。
5. 问答能力：基于检索上下文调用 AI 模型生成回答。
6. 前端界面：实现文档上传、检索输入和问答结果展示。
7. 测试与文档：补充单元测试、集成测试和使用说明。
